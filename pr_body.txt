### Overview
This pull request brings full responsiveness, seamless layout scaling, and visual polish to the dashboard sidebar. It resolves layout jitter on desktop viewports by nesting the sidebar back into the scaled canvas, replaces the generic hamburger toggle with the raccoon icon, configures a scrollable drawer layout for mobile viewports, and adjusts spacing gaps to segment bottom-level actions cleanly.

It also refactors the dashboard page by extracting all sidebar UI into a dedicated component, so layout scaling and main content stay in `page.tsx` while navigation chrome lives in one reusable module.

### Key Changes
1. **Desktop Sidebar Canvas Integration**
   - Renders the sidebar inside the scaled `.dashboard-canvas` container using absolute positioning for screen widths ≥ 768px.
   - Restores the layout structure from PR #196 to inherit browser-native scaling transforms, eliminating lag and jitter.

2. **Responsive Scrollable Mobile Drawer**
   - Configures a fixed mobile sidebar drawer (`width: 280px`) on viewports < 768px with a backdrop overlay.
   - Splits the mobile drawer into a fixed header/divider and a scrollable nav-items container (`flex: 1`, `overflow-y: auto`) to accommodate short screen heights (e.g., 375×608px).
   - Prevents header/button cutoffs by allowing users to scroll naturally.

3. **Raccoon Logo Toggle Button**
   - Replaced the separate hamburger SVG icon with the raccoon logo button acting as the toggle controller.
   - Fits canvas coordinates seamlessly on desktop, fading out when the sidebar is open and showing inside the header to close. On mobile, it displays as a fixed floating toggle button.

4. **Visual Layout and Spacing Improvements**
   - Created clean 40px vertical gaps dividing the profile navigation menu, **Log Out** button, and **Need Help** documentation card.
   - Fixed a bug where unmounting the desktop sidebar on mobile caused top-divider strokes to render transparent by moving gradient SVG definitions local to each viewport's sidebar.

5. **Sidebar Component Extraction** (`components/dashboard/dashboard-sidebar.tsx`)
   - Moved ~600 lines of sidebar markup from `app/dashboard/page.tsx` into `DashboardSidebar`.
   - `page.tsx` now owns only viewport scaling, open/close state, and main content; it renders `<DashboardSidebar mode="mobile" />` outside the canvas and `<DashboardSidebar mode="desktop" />` inside it.
   - Preserves existing design, breakpoints, animations, and nav structure with no visual or responsive changes.
   - Consolidates logout handling, pathname-based active states, and the need-help/docs card inside the sidebar component.
